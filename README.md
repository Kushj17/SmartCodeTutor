# AI Code Tutor: Multi-Agent Educational Code Explainer

## Overview
This project is an AI-powered educational tool designed to help users understand Python code snippets. It leverages a fine-tuned local CodeT5 model to generate a baseline technical explanation of the provided code, and a multi-agent system powered by CrewAI and Gemini to adapt the explanation based on the user's expertise level (Beginner, Intermediate, Advanced).

## Features
- **Local CodeT5 Model:** A fine-tuned sequence-to-sequence model that provides an initial technical analysis of the code.
- **CrewAI Multi-Agent Architecture:**
  - **Technical Code Analyst:** Uses the local CodeT5 model to extract meaning.
  - **Educational Personalization Expert:** Simplifies the technical explanation based on the selected user level.
  - **Algorithm Specialist:** Provides Big O Time and Space complexity analysis.
  - **Content Quality Lead:** Formats the final explanation into a clean, structured Markdown report.
- **Streamlit Interface:** An easy-to-use web interface to paste code, select expertise level, and view the generated explanation.
- **Jupyter Notebooks:** Included notebooks (`fine_tuning.ipynb`, `phase1.ipynb`, `agent.ipynb`) used for dataset preparation, model fine-tuning, and agent experimentation.

## Prerequisites
- Python 3.10+
- An active Gemini API Key

## Setup & Installation

1. **Clone the repository** (if applicable) or navigate to the project directory:
   ```bash
   git clone https://github.com/Kushj17/SmartCodeTutor.git
   ```

2. **Install the dependencies:**
   It is recommended to use a virtual environment.
   ```bash
   pip install -r requirements.txt
   ```

3. **Set your Gemini API Key:**
   The Streamlit app requires a Gemini API Key. You can set it in your environment or update the `src/app.py` file directly (ensure you do not commit sensitive keys to public repositories).
   
   ```python
   # In src/app.py
   import os
   os.environ["GEMINI_API_KEY"] = "YOUR_GEMINI_API_KEY"
   ```

4. **Ensure the local model is available:**
   The app expects the fine-tuned CodeT5 model to be located in the `./codet5-beginner-explainer` or `./final_codet5_model` directory (or wherever your notebook outputs the model inside `results/` or similar folders). If you haven't fine-tuned it yet, run the `notebooks/fine_tuning.ipynb` notebook to generate the model checkpoints.

## Running the Application

To start the Streamlit web interface, run the following command in your terminal from the root directory:

```bash
streamlit run src/app.py
```

This will launch the application in your default web browser. 

## Project Structure

- `src/app.py`: The main Streamlit application script containing the UI and CrewAI agent logic.
- `requirements.txt`: List of required Python packages to run the project.
- `notebooks/fine_tuning.ipynb`: Notebook for fine-tuning the base CodeT5 model using the provided dataset.
- `notebooks/agent.ipynb`: Notebook containing testing and experimentation with the CrewAI agent setup.
- `notebooks/phase1.ipynb`: Initial notebook for project setup or earlier phases of development.
- `data/dataset.json` / `data/dataset_fixed.json`: The dataset files used for fine-tuning the CodeT5 model.
- `results/`: Directory intended for containing the fine-tuned model checkpoints and other outputs generated by notebooks.

## How it Works
1. The user inputs a Python code snippet and selects their proficiency level.
2. The Streamlit app kicks off a CrewAI process.
3. The **Technical Code Analyst** agent uses a custom `LocalCodeT5Tool` to query the local, fine-tuned CodeT5 model.
4. The output is passed to the **Educational Personalization Expert** and **Algorithm Specialist** (powered by Gemini) to simplify the explanation and analyze algorithmic complexity.
5. Finally, the **Content Quality Lead** compiles everything into a cohesive, readable format and displays it on the UI.
